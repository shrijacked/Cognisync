# Cognisync v0.1.4

This release turns Cognisync into a more disciplined operator system.

## Highlights

- added stable source, graph, review, and run manifests so corpus state persists cleanly between passes
- deepened the research runtime with resumable plans, stricter citation and conflict validation, and explicit answer modes
- turned graph intelligence into an actionable operator loop with review queues, deterministic maintenance, dismiss/reopen controls, and corpus change summaries
- added maintenance policy controls in workspace config plus one-off CLI overrides so teams can tune how aggressively concept pages are accepted
- taught `doctor` to surface maintenance policy health alongside workspace and adapter readiness

## Validation

- `python3 -m unittest discover -s tests -v`
- `/usr/bin/env PYTHONPYCACHEPREFIX=/tmp/cognisync-pyc python3 -m compileall src tests`
- `PYTHONPATH=src python3 -m cognisync review reopen --help`
- `PYTHONPATH=src python3 -m cognisync maintain --help`
